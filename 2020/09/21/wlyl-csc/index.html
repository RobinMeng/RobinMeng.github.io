    TCP提供服务的主要特征

    1. 面向连接的传输
    2. 端到端的通信，不支持广播通信
    3. 高可靠性，确保传输数据的正确性，不出现丢失和乱序
    4. 全双工方式传输
    5. 采用字节流方式，以字节为单位传输
    6. 提供紧急数据传输功能

    UDP提供服务的特征

    1. 无连接传输
    2. 不对数据进行检查和修改
    3. 无须等待对方应答
    4. 因以上特征 所以效率高传输快
    5. 无连接 不检查 不等待 效率高

    什么是传输服务

    1. 位于网络层和应用层之间
    2. 利用网络层提供的服务向应用层提供服务
    3. 协议不同分为 面向连接 和 非连接 两种类型

    TCP 重传策略

    1. 判定数据段是否需要重传的依据是设置重发定时器
    2. 发送一个数据段启动一个重发定时器
    3. 如果在定时器超时前收到确认，就关闭定时器
    4. 如果在定时器超时前未收到确认 就重传该数据段

    TCP 与UDP 的不同

    1. TCP 需要建立连接 UDP 无须建立建立连接
    2. TCP 应用于大数据量传输 UDP 应用于一次只传少量数据
    3. TCP具有高的可靠性 UDP应用程序需要负责可靠
