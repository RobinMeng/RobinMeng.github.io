    什么是应用层

    应用层是网络协议中的一层 最高的一层 基于传输层

    应用层的知识结构

    1. 域名系统
    2. 电子邮件
    3. 万维网
    4. 其他服务

    域名系统

    1. IP地址由网络标识 和 主机标识 两部分组成
    2. 域名分为 网络域名 和 主机域名
    3. 我国将二级域名划分为 类别域名 和 行政区域域名

    Email

    1. 用户Email地址格式为 用户名@主机域名
    2. SMTP 服务器基于DNS中的邮件交换记录来路由电子邮件
    3. SMTP 通过用户代理程序 和 邮件传输代理 实现邮件传输
    4. IMAP 提供离线 在线 断开连接 三种工作方式

      POP3 收发邮件过程

    5. 通过POP3 客户端程序登陆到支持POP3协议的邮件服务器，用户可以发送邮件及附件
    6. 邮件服务器将该用户收存的邮件传递给POP3的客户群，使用户收到这些邮件，并将这些邮件从服务器上删除
    7. 邮件服务器再将用户提交发送邮件，转发到运行SMTP协议的计算机中，通过它实现邮件的最终发送

    IMAP的断连工作方式

    1. 断连是一种脱机与联机操作的混合模式
    2. 客户机与邮件服务器建立连接，客户端邮件程序选取相关邮件，在本地生成告诉缓存，同时 选取的邮件仍保存在邮件服务器上
    3. 断开与邮件服务器的连接，对客户机高速缓存进行脱机处理，再次连接时，进行同步处理使邮件状态相一致

    IMAP4的特点

    1. 提供离线 在线 断开三种工作方式
    2. 支持 脱机 联机 和 断连操作
    3. 用户可通过浏览器决定是否要下载 删除 检索信件的特定部分，还可以在服务器上修改创建文件夹或邮箱

    WWW

    1. www 采用的是C/S 工作模式
    2. URL 是统一资源定位符

    远程登陆Telnet过程

    1. 本地启动Telnet客户端应用程序
    2. 远程机器拥有Telnet 服务端程序
    3. 客户端程序指定远程机器的名称，通过Internet 建立连接
    4. 连接建立成功后就像访问本地计算机一样可以访问远程计算机资源
    5. 退出后控制权回到本地计算机

    WWW 的工作过程

    1. 客户端建立连接，用户通过浏览器向Web服务器发送浏览信息请求
    2. web服务器接收到请求，向浏览器返回请求信息
    3. 关闭连接
