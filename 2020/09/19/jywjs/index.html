    局域网技术

    介质访问控制

    1. 信道访问权分配 静态分配 动态分配
2. 介质访问控制协议 争用协议 无冲突协议 有限争用协议

    简述时分ALOHA 基本思想 – 介质有冲突控制协议

    1. 将时间分成等长间隙 每个间隙可以发送一个帧
2. 用户有数据帧发送时 不论帧在何时产生 必须等到下一个时隙才能发送
3. 用户通过监听信道来获知是否产生冲突，数据传输是否成功，若发现失败等待若干间隙后重新发送

    简述二进制退避算法

    1. 对每个数据帧，当第一次发生冲突时 设置一个参量L=2
2. 退避取1到L个时间片中的一个随机数
3. 当数据帧再次发生冲突时，L翻倍
4. 设置一个最大的重传次数，超过次数不再重传，并报告错误

    IEEE 与局域网

    1. 预约是各个网络节点首先声明自己有数据要发送，然后根据声明的顺序获得信道的使用权来发送数据
2. 网络负载 是指单位帧时内系统发送所有数据帧的平均数量
3. 吞吐量 指单位帧内系统成功发送新产生的数据帧的平均数量
4. 在FDDI环网中采用弹性缓冲器来消除时钟偏移现象
5. IEEE MAC帧的起始界定 10101011

    令牌环操作过程

    1. 网络空闲时 只有一个令牌在环路上绕行
2. 当一个站点要发送数据时，必须等待并获取一个令牌，将令牌标志为1，随后发送数据
3. 环路上的每个站点边转发数据，边检测数据帧中的目的地址，若为本站地址读取携带的数据
4. 数据帧环绕一周返回时，发送站将其从环路上撤销，同时根据返回的有关信息确定数据是否出错。若有错重发存于缓冲区中的待确认帧，否则释放缓冲区中的待确认帧
5. 发送站点完成数据发送后，重新产生一个令牌传至下一个站点，以便其他站点使用

    4. 优先权选择
