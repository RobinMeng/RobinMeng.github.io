    关系数据库

    数据库原理
    发布时间 : 2020-10-13 14:07
    字数:473 阅读 :

    2. 关系数据库对关系的要求
    3. 冗余和异常问题
    4. 关系规范化过程
    5. 关系模型完整性约束的检验
    6. 什么是完全函数依赖

    知识图谱

    1. 关系模型的三个组成要素是关系数据库结构 关系操作集合 和 关系完整性约束
    2. 在一个关系R中，若每个数据项都是不可再分的原子项，那么R 属于 1NF
    3. 数据库的数据完整性指的是数据库中数据的 正确性 相容性 一致性

    关系数据库的基本特征

    关系数据库的基本特征是使用关系数据模型组织数据

    关系数据库对关系的要求

    1. 每一个属性都不可在分解
    2. 每一个关系仅仅有一种关系模式。
    3. 每一个关系模式中的属性必须命名，在同一个关系模式中，属性名必须是不可分解的
    4. 同一个关系中不允许出现候选码或候选健完全相同的元组
    5. 在关系中元组的顺序是无关紧要的 可任意交换
    6. 在关系中属性的顺序是无关紧要的

    冗余和异常问题

    1. 数据冗余
    2. 更新异常
    3. 插入异常
    4. 删除异常

    关系规范化过程

    1. 一个低一级范式的关系模式 通过模式分解转换为若干个高一级的关系模式的集合的过程 称为 规范化
    2. 在关系数据库系统中 所有的关系结构都必须规范化 至少是第一范式

    关系模型完整性约束的检验

    对关系数据库执行插入，更新 删除操作时，检验是否满足实体完整性约束，参照完整性约束，用户自定义完整性约束

    什么是完全函数依赖

    设R为一给定的关系， X Y 为其中属性集，若X->Y,且 对X中的任何真子集 X， 都有 X，-> != 则称Y完全函数依赖X
