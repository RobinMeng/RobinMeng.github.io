    知识图谱

    1. 数据库设计的目标包括 满足应用功能需求 和 良好的数据库性能
    2. 基于第三范式的设计方法是一种 结构化 的设计方法
    3. 需求分析 的目标是了解与分析用户的信息及应用处理的需求，并将结果按一定格式整理而形成需求分析报告
    4. 数据字典 的功能是存储和检索各种数据描述
    5. 实体分析 又称为 自顶向下法
    6. 加载数据时 收集 分类 和 整理 由 人工 完成
    7. 关系数据库的概念结构设计通常采用 自顶向下法
    8. 实体选择时最大的困难是 如何区分实体与属性
    9. 同名异义 指的是不同意义的实体类型名或联系类型名在不同的局部应用中具有相同的名字

    什么是数据库设计

    1. 数据库系统与现实世界进行密切 有机 协调一致集合的过程

    数据库设计方法的分类

    1. 直观设计
    2. 规范设计
    3. 计算机辅助设计

    什么是计算机辅助设计

    1. 以领域专家的知识或经验为主导
    2. 模拟某一规范化的设计方法
    3. 通过人机交互的方式来设计某些过程

    需求分析的步骤

    1. 确定数据库的范围
    2. 分析数据应用过程
    3. 收集与分析数据
    4. 编写需求分析报告

    什么是数据字典

    1. 是数据库系统存储三级结构定义
    2. 数据库系统中各类数据详细描述的集合

    需求分析报告的内容、

    1. 数据库的应用功能目标
    2. 标明不同用户视图范围
    3. 应用处理过程需求说明
    4. 数据字典
    5. 数据量
    6. 数据约束

    简述概念结构的设计任务

    1. 在需求分析报告的基础上 设计满足应用需求的用户信息结构
    2. 该信息结构通常称为概念模型

    简述逻辑结构设计包含那些输入信息

    1. 独立于DBMS的概念模型
    2. 有关响应时间 安全保密性 数据完整性及恢复方面的要求说明 保持数据一致性的规则说明
    3. 数据量及使用频率
    4. 特点 DBMS 特性 包括 DBMS支持的数据模型及数据定义说明

    逻辑结构设计包含的输出信息

    1. 一个特定DBMS支持的概念模式
    2. 一个或多个外部视图
    3. 物理设计说明 包括存入数据库中的数据量 使用频率及响应时间要求
    4. 程序设计说明

    设计评价的方法

    1. 通过程序设计指南提交程序执行逻辑步骤在子模式上的模拟执行来考核模式及自模式是否满足应用需求
    2. 估计数据容量及存取效率
    3. 为物理设计提供参考信息
    4. 发现不合理之处 返回模型转换处重新执行，如此反复 直到满足要求为止

    简述物理设计的任务

    1. 确定数据库在存储设备上的存储结构及存取方法
    2. 确定建立索引和聚集 以及物理大小 缓冲区个数大小 数据压缩的选择等

    数据库实施阶段要完成的工作

    1. 加载数据
    2. 应用程序设计
    3. 数据库试运行

    概念结构设计步骤

    1. 建立局部信息结构
    2. 将局部信息结构合成全局信息结构 并优化

    什么是一对多联系

    对于实体集A中的每一个实体，实体集B中都有N个实体与之联系 称实体集A 与 实体集B 具有一对多的联系 1:N

    概念结构设计确定局部范围的准则

    1. 部门和功能相对独立
    2. 其他局部范围相互影响较少
    3. 实体个数适中

    简化数据模型的优化方法

    1. 确定各属性间函数依赖关系
    2. 消除冗余
    3. 根据实际需求确定合适的范式
    4. 确定是否需要对某些模式进行合并或分解
    5. 进行必要的分解，提高数据操作的效率和存储空间的利用率
