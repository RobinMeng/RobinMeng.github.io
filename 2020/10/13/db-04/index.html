    SQL与关系数据库的基本操作

    数据库原理
    发布时间 : 2020-10-13 14:07
    字数:958 阅读 :

    2. SQL的组成部分
    3. 数据库操作语言

    6. 在Mysql中子查询有几类

    知识图谱

    1. 从20世纪80年代以来 SQL 一直是关系数据库关系系统的标准语言
    2. 数据定义语言用于对数据库及数据库中的各种对象进行 创建 删除 修改 等操作
    3. 数据库对象主要有 表 默认约束 规则 视图 触发器 存储过程等
    4. 数据库操作语言主要用于操纵数据库中的各种对象，特别是 检索 和 修改对象 数据
    5. 嵌入式和动态sql规划 规定了SQL语句在高级程序设计语言中使用的规范方法，以便适应更为复杂的应用
    6. 数据控制语言主要用于 安全管理
    7. SQL调用包括 SQL例程 和 调用规划 以便提高SQL的灵活性，有效性 共享性 以及使用sql 具有更多的高级语言特性
    8. 目前，使用mysql 数据库管理系统构建各种信息管理系统或互联网站应用环境，主要有 LAMP 和 WAMP 两种构建方式
    9. 数据库中只存储视图的定义而不存储视图对应的数据，这些数据仍存储在导出视图的基本表中，因此 视图是一个 虚表
    10. 常量 是指在程序运行过程中值不变的量
    11. 字符串常量分为 ASCII 字符串 常量 和 Unicode 字符串常量
    12. 数值常量可以分为 整数常量 和 浮点数常量
    13. NULL值 通常用于表示 没有值 无数据 等意义
    14. 表达式 是常量 变量 列名 复杂计算 运算符 和 函数组合
    15. SQL的数据定义功能包括数据库 模式定义 表定义 索引定义 和 视图定义
    16. 在Mysql中 可以使用 show databases 或 show schemas 语句查看可用数据库列表
    17. 数据表 是关系数据库中最重要 最基本的对象数据
    18. 如果删除数据库中已经存在的表，可以通过使用 drop table 语句来实现
    19. 在数据库中 查看表包括 显示表名称 显示表结构 两种情形
    20. 在mysql中 使用use 语句 可以实现从一个数据库 跳转到另一个数据库
    21. 表中的索引越多 则更新表的时间就会越长
    22. 在实际使用中 索引通常被创建成 单列索引 和 组合索引
    23. 当一个索引不再需要时，可以使用drop index 语句 或者 alter table 语句来进行删除
    24. 如果需要查看表中创建索引的情况 可以使用 show index 语句
    25. 数据查询 是sql语言的核心功能 也是数据库中使用最多的操作
    26. sql 提供了select 语句进行数据查询 提供 选择 投影 和 连接
    27. count 函数用于求组中 返回int类型整数
    28. sum 函数 功能是返回表达式中所有值的和
    29. variance 函数的功能是返回给定表达式中所有值的方差

    SQL的特点

    1. SQL 不是某个特定数据库供应商的专用语言
    2. SQL简单易学
    3. SQL 灵活 强大的语言
    4. 不区分大小写

    SQL的组成部分

    1. 数据定义语言DDL
    2. 数据操作语言DML
    3. 数据控制语言DCL
    4. 嵌入式和动态SQL规划
    5. SQL调用和会话规则

    数据库操作语言

    1. select
    2. insert
    3. update
    4. delete

    使用mysql 应用环境架构的方式

    1. LAMP
    2. WAMP

    在Mysql 中创建索引的方式

    1. create index
    2. create table
    3. alter table

    在Mysql中子查询有几类

    1. 表子查询
    2. 行子查询
    3. 列子查询
    4. 标量子查询

    简述视图与表的区别

    1. 表是真实物理表 视图是虚拟表
    2. 表数据真实存在 视图诗句引用动态生成
    3. 视图数据引用的是真实表数据
